### üìã Overview
This repository also contains a Postman collection that targets the public Restful Booker API. The collection exercises basic CRUD operations and token-based authentication to validate request/response behavior and data consistency.

The API base URL used in the collection is: `https://restful-booker.herokuapp.com`

### üéØ Objectives
- Validate different HTTP methods (`GET`, `POST`, `PUT`, `PATCH`, `DELETE`).
- Check correct request/response handling and data consistency.
- Perform token-based authorization testing.

## üîÅ How to Import & Run the Collection

### Import into Postman (GUI)
1. Open Postman.
2. Click "File" ‚Üí "Import" and select the collection file `restful-booker-collection.json` (or drag-and-drop it into the Import dialog).
3. Optionally import an environment JSON to set base URL and any variables (if provided).
4. Open the collection and use the Collection Runner to run all requests or individual folders. Configure iterations, data files, and environment before running.

### Run with Newman (CLI) ‚Äî PowerShell examples

Using `npx` (recommended, no global install):

```powershell
# Run the Restful Booker collection and export an HTML report to newman/
npx newman run .\restful-booker-collection.json -e .\restful-booker-environment.json -r cli,htmlextra --reporter-htmlextra-export .\newman\restful-booker-report.html
```

If you have `newman` installed globally:

```powershell
newman run .\restful-booker-collection.json -e .\restful-booker-environment.json -r cli,htmlextra --reporter-htmlextra-export .\newman\restful-booker-report.html
```

Notes:
- If you don't have an environment file, you can pass the base URL directly in Postman or add it to the collection variables.
- For authenticated endpoints (PUT/PATCH/DELETE), first call `POST /auth` with the correct credentials to obtain a token, then include that token in the `Cookie` or `Authorization` header as required by the API.

### Add an npm script (optional)
Add this to `package.json` under `scripts` to make running easier:

```json
"scripts": {
  "test:restful-booker": "newman run restful-booker-collection.json -e restful-booker-environment.json -r cli,htmlextra --reporter-htmlextra-export newman/restful-booker-report.html"
}
```

Run with:

```powershell
npm run test:restful-booker
```

---

### Troubleshooting
- Ensure Node.js and npm are installed.
- Run `npm install` if you add or change dependencies (e.g., `newman-reporter-htmlextra`).
- If a request fails, verify the environment variables (base URL, auth credentials) and try the request manually in Postman to inspect headers and body.

# Heroku API Test

This repository contains a Postman collection and environment for testing a Heroku-hosted API, plus Newman report artifacts.

Files of interest
- `HerokuApp.postman_collection.json` ‚Äî Postman collection containing the API tests.
- `HerokuApp.postman_environment.json` ‚Äî Environment file for the collection (variables like base URL, auth, etc.).
- `newman/` ‚Äî Folder containing HTML reports generated by Newman. Example: `newman/HerokuApp-2025-11-12-07-55-56-603-0.html`.
- `package.json` ‚Äî Lists dependencies used for reporting (e.g., `newman-reporter-htmlextra`).

Prerequisites
- Node.js and npm (or npx).
- Newman (Postman's command-line runner). You can install it globally or use `npx` to run it without global install.

Recommended reporters
- The project includes `newman-reporter-htmlextra` as a dependency to produce richer HTML reports. You can use it with Newman by passing `--reporters htmlextra` and configuring the output path.

Run the collection (PowerShell examples)

Using npx (no global install required):

```powershell
# Run collection with CLI and HTML Extra reporter and save HTML to newman/ folder
npx newman run .\HerokuApp.postman_collection.json -e .\HerokuApp.postman_environment.json -r cli,htmlextra --reporter-htmlextra-export .\newman\HerokuApp-report.html
```

Using a globally installed newman:

```powershell
# If you have newman installed globally
newman run .\HerokuApp.postman_collection.json -e .\HerokuApp.postman_environment.json -r cli,htmlextra --reporter-htmlextra-export .\newman\HerokuApp-report.html
```

Notes and tips
- If you want to keep multiple reports, add a timestamp to the export filename (PowerShell can help generate timestamps).
- You can open the generated HTML reports in your browser (e.g., double-click the file in Explorer or use `start .\newman\HerokuApp-report.html`).

Adding an npm script (optional)
If you'd like an npm command to run the collection, add a script to `package.json` such as:

```json
"scripts": {
  "test:newman": "newman run HerokuApp.postman_collection.json -e HerokuApp.postman_environment.json -r cli,htmlextra --reporter-htmlextra-export newman/HerokuApp-report.html"
}
```

Then run:

```powershell
npm run test:newman
```

Troubleshooting
- If the report doesn't generate, ensure `newman-reporter-htmlextra` is installed: `npm install`.
- If Node/npm or npx cannot be found, install Node.js from https://nodejs.org/.
